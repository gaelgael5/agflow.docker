Où peut-on lâcher les fichiers dans la page Role ?

Trois mécaniques d'UX possibles. La page actuelle a une sidebar à gauche (arborescence sections + documents) et un panneau droit (onglets general / prompt / document). Choisis une seule option.

A

Drop sur chaque section dans la sidebar (ciblage précis)

Quand tu drag, chaque section ("MISSIONS", "COMPETENCES", ou tes sections custom) se highlight séparément. Tu lâches précisément sur la section voulue.

Mockup — highlight par section
📁 ROLES
  └─ role.md
📁 MISSIONS ← drop ici
  └─ m1.md
📁 COMPETENCES
  └─ c1.md
📁 CUSTOM-1
Panneau droit
(onglets)

✓ Cible claire · ✓ pas d'ambiguïté · ✗ zones de drop étroites (plus difficile de viser)

B

Drop sur le panneau droit (section = celle actuellement ouverte)

Tu cliques d'abord sur une section pour l'ouvrir (normal), puis tu drop n'importe où dans le panneau principal de droite. Le contexte (section active) définit la destination.

Mockup — drop dans le panneau actif
📁 ROLES
📁 MISSIONS ← active
📁 COMPETENCES
📁 CUSTOM-1
↓ Lâche des fichiers ici
(destination: MISSIONS)

✓ Grande zone facile à viser · ✗ destination implicite (risque de se tromper)

C

Overlay plein-écran avec sélecteur de section

Dès que tu drag un fichier au-dessus de la page, un overlay s'affiche avec la liste des sections disponibles. Tu choisis au moment du drop.

✓ Explicite, zéro ambiguïté · ✗ un clic de plus · ✗ plus lourd à implémenter

Ma reco : A. C'est le comportement standard (Gmail, macOS Finder, VS Code Explorer). La zone est assez grande si on hover aussi la ligne entière et les documents enfants. Simple à implémenter avec react-dropzone par section.
