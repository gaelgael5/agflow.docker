Quel feedback visuel pendant/après le drop ?

Quatre moments à designer : (1) hover initial (fichier au-dessus de la page), (2) hover sur une section cible, (3) pendant upload, (4) après succès/échec. Choisis une ambiance.

A

Sobre & inline — highlight vert + toast après

Section cible : bordure et fond verts dashed. Pas d'overlay. Les autres sections restent visibles en gris neutre. Toast shadcn (sonner) en bas-droit après succès/échec.

Pendant hover sur MISSIONS
📁 ROLES
📁 MISSIONS ← drop 2 fichiers ici
📁 COMPETENCES
📁 CUSTOM-1

Après succès : toast "✓ 2 documents créés dans MISSIONS"

B

Overlay modal — liste des fichiers + progress

Dès le hover initial, un petit panneau modal s'ouvre à côté de la sidebar, listant les fichiers en cours d'upload avec une barre de progression par fichier. Le panneau reste après le drop pour montrer le résultat final.

Modal de progression
Import dans MISSIONS

📄 mission-a.md   ✓ créé
📄 mission-b.md   ■■■■■□□ 60%
📄 mission-c.md   ⏳ en attente

Avantage : l'utilisateur voit exactement où en est le batch · Coût : plus de code UI

C

Minimaliste — juste un spinner inline + toast final

Pendant le hover : juste le curseur change et un liseré vert discret sur la section. Pendant l'upload : spinner à côté du titre de section. Après : toast résumé unique.

Pendant upload
📁 ROLES
📁 MISSIONS ⟳ (upload 2/3)
  └─ existing.md
📁 COMPETENCES

Le plus léger · l'utilisateur ne voit pas chaque fichier, juste le total

Ma reco : A. Cohérent avec le reste de la page (déjà basée sur shadcn + sonner). Pour un batch normal (2-5 fichiers .md), pas besoin d'un panneau de progression (B). Si un jour tu imports 50 fichiers, on réévalue.
