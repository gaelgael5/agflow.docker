Drag-and-drop dans les sections du Rôle — quel comportement ?

Aujourd'hui les sections ("roles", "missions", "competences" + sections custom) contiennent des documents markdown texte. Le drop peut avoir plusieurs sens. Coche tout ce qui t'intéresse (plusieurs combinables).

A

Drop .md / .txt → nouveau document

Tu glisses un fichier texte depuis l'explorateur → le contenu devient un nouveau document markdown dans la section ciblée. Nom du fichier = nom du document. (Cas simple, colle avec le modèle actuel)

Avant / Après
📄 mission-audit.md
(sur le bureau)
→
📁 MISSIONS
├─ existing.md
└─ mission-audit.md ✨
B

Drop fichier binaire (PDF, image, doc) → attachment stocké

Tu glisses un PDF, une image, un .docx → stocké côte à côte avec les documents markdown. Nouveau type de ressource. (Demande migration DB : nouvelle table role_attachments, storage de blob)

Nouvelle nature de ressource
📎 charte-rh.pdf
🖼 logo.png
→
📁 COMPETENCES
├─ 📄 text-doc.md
├─ 📎 charte-rh.pdf
└─ 🖼 logo.png
C

Drag document existant entre sections → déplacement

Tu attrapes un document déjà dans la sidebar et tu le lâches dans une autre section pour le reclasser. Pas de fichier externe, juste réorganisation. (UPDATE role_documents SET section=... — endpoint manquant)

Réorganisation interne
📁 ROLES
└─ 📄 doc-A.md (drag)
→
📁 MISSIONS
└─ 📄 doc-A.md ✨
D

Drop multi-fichiers → import batch

Tu sélectionnes plusieurs fichiers dans l'explorateur et tu les lâches d'un coup dans une section. Tout est créé en une seule opération. (Orthogonal à A/B — c'est la multiplicité, pas le type)

Batch import
📄 a.md 📄 b.md 📄 c.md
→
📁 ROLES
├─ a.md ✨
├─ b.md ✨
└─ c.md ✨

Ma recommandation : commencer par A + D (drop .md/.txt en mono ou multi) — collait au modèle actuel, aucune migration DB. Puis C si tu veux la réorg. B ouvre un chantier plus lourd (storage de blobs).
